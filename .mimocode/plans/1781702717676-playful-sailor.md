# Plan: Ampliar Base de Datos + Mejoras Visuales — ItRunsOnLinux

## Contexto del Proyecto
- **Stack**: Astro 6.x SSR en Cloudflare, D1 database (Drizzle ORM), i18n EN/ES
- **DB actual**: ~1250+ apps en `seed-bulk.sql`, categorías existentes: Utilities (432), Development (236), Design (143), Games (121), Audio (99), Office (79), Social (39), Windows Integrated (27), Engineering (16)
- **Categorías faltantes**: Security/Networking, Photography/Video, Education/Finance
- **Constraint**: DB en producción, sin cambios de schema, sin duplicados, usuario ejecuta wrangler commands

---

## Riesgos y Mitigaciones

| Riesgo | Mitigación |
|--------|------------|
| Duplicados en DB | Script check-duplicates.js antes de ejecutar |
| Romper producción | Solo INSERTs, nunca UPDATE/DELETE en datos existentes |
| Performance con más apps | SearchBar ya carga todo client-side; 200 apps más es manejable |
| i18n roto | Verificar todas las keys nuevas en ambos idiomas |
| CSS roto | Probar en dark y light mode |
